Set up your Linea Testnet RPC node

## Prerequisites

- [Setup your Axelar validator](/validator/setup)
- Minimum hardware requirements: CPU with 2+ cores, 1GB RAM.
- MacOS or Ubuntu 18.04+
- [Official Documentation](https://docs.linea.build/developers/useful-info)


## Steps
1. Get Linea RPC
2. Configure vald

### Get Linea RPC

Linea's p2p network is currently permissioned. Hence, we'll use the public/third-party RPC
for the testnet integration.

To test your Linea RPC node, you can send an RPC request using `curl`.

```bash
curl -X POST --url [rpc] \
  -s -H "Content-Type: application/json" \
  --data '{"jsonrpc":"2.0","method":"eth_syncing","params":[],"id":1}'
```

### Configure vald

In order for `vald` to connect to your Linea node, your `rpc_addr` should be exposed in
vald's `config.toml`


<Tabs items={['Testnet']}>
  <Tab>
  ```bash
  [[axelar_bridge_evm]]
  name = "linea"
  rpc_addr = "http://IP:PORT"
  start-with-bridge = true
  ```
